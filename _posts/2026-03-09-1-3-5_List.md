What I built this week (WIBTW).

&nbsp;
My starting prompt: "A long time ago I used this very simple web app for a 1-3-5 to-do list approach. It was a simple site that set 1 big, 3 medium and 5 small tasks, and was essentially just so you could edit then make it a check box. Like in Apple notes tbh. Is it easiest just to use Apple notes to do this on my phone, or how hard would it be to run some sort of webapp accomplishing this. I have a domain but no server (other than GitHub serving static pages), but than maybe streamlit or shiny… but I don’t think they are persistent."

&nbsp;  
GitHub Pages (static HTML/JS) — easiest, free, already have it. A single HTML file with localStorage can save your tasks persistently in the browser. The catch: it's per-device, so your phone and laptop won't sync. But honestly for a daily to-do reset tool, that might be fine.

&nbsp;
This is a pure static HTML file, no backend needed. I'll build it now and you can drop it straight into your GitHub Pages repo.

&nbsp;  
It's a single 135-todo.html file — just drop it into your GitHub Pages repo. Here's what it does:

- Click to check tasks off, color-coded by tier (red / green / navy)
- Click any text to edit inline — tasks are saved instantly to localStorage so they persist across browser sessions on that device
- Progress bar per section fills as you check things off
- "Clear all checks" button at the bottom resets checkmarks but keeps your task text (so you can reuse the same list tomorrow without retyping)


&nbsp;

I later did minor modifications, but basically this was the first demo of a static html app that uses local memory.
